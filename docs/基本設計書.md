# YGO Combo Maker - 基本設計書 (Basic Design)

## 1. システムアーキテクチャ
本システムはフロントエンド単体（HTML / CSS / Vanilla JavaScript）で構築されるSPA (Single Page Application) に準ずるアーキテクチャを持つ静的サイトである。サーバーサイドアプリケーションや外部データベースを一切必要とせず、ユーザーの端末内（ブラウザ）でセキュアかつ高速に稼働する。

## 2. ディレクトリ構成
* `index.html`: アプリケーションのベースとなるDOMツリー、およびモーダルUI（インポート、画像切り抜き用ウィンドウ）を定義。
* `style.css`: CSS Variablesを利用した一元的なカラーテーマ、ならびにFlexbox機能やCSS Grid、Absolute Positioning（絶対座標制御）を複雑に組み合わせたスクロール保証型レイアウト。
* `app.js`: アプリケーションの全ロジック（DOMとステートのバインディング、イベントハンドラ、APIからの非同期取得、Canvas操作による画像処理）を統合したスクリプト。

## 3. UI/UX設計（画面レイアウト）
画面は大きく以下の2ペイン（分割領域）で構成され、モバイル幅（768px未満）では流動的に上下にスタックされる：

* **Canvas Area（左側展開エリア）**:
  クリックされたカードやアクションが順番に並べられるメイン描画エリア。ルートの視覚化と保存を目的とする。
* **Palette Area（右側操作エリア）**:
  スクロール不全バグを起こさないよう絶対座標(`position: absolute`)で制御された3形態のタブコンテナ。
  1. `デッキ`: ユーザーが現在構築中のデッキカード・ならびに作成されたオリジナル画像カードの表示・管理・削除を行うベース領域。
  2. `追加検索`: YGOPRODeck APIを利用したカード検索とパレットへの送信プール。
  3. `アクション`: 「通常召喚」「サーチ」などの行動ノード一覧。

## 4. データフロー設計
* **API連携**: アプリケーションロード時に `https://db.ygoprodeck.com/api/v7/cardinfo.php?language=ja` から約13,000枚のJSONデータを取得。3日間（72時間）のTTL付きでブラウザの `LocalStorage` にキャッシュ・圧縮格納し、過剰なAPIリクエスト制限とロード時間のオーバーヘッドを回避する。
* **永続化とオートセーブ**: ユーザーがカードの追加、削除、デッキ作成等のあらゆる操作を行うたびに、複数デッキの状態やタイトルを随時 `LocalStorage` に即時同期させ、リロード時にも瞬時に前回の状態へ復帰する。
