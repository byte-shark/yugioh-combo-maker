# YGO Combo Maker - 定数設計書 (Constants Design)

## 1. LocalStorage キー定義
ブラウザのローカル環境に保存、および参照されるデータのキー一覧。
* `ygo_multiple_decks` : 全てのデッキプロファイル情報を格納するJSONのマスター元文字列。
* `ygo_current_deck_id` : 前回起動時に最後に開いていたデッキのID文字列。
* `ygo_combo_title` : キャンバス上部に記入された展開ルートのヘッダータイトル。
* `ygo_db_ja` : YGOPRODeckから取得した日本語カードデータベースのキャッシュ実体。
* `ygo_db_time` : データベースをキャッシュした際の日時（Unixタイムスタンプ）。起動時に取得して現在時刻と比較し、無駄なAPI取得フェーズをスキップするための計算基準として利用。

## 2. 外部APIエンドポイント定数
* **YGOPRODeck JSON API**: `https://db.ygoprodeck.com/api/v7/cardinfo.php?language=ja`
  遊戯王カードの基本データ（ID、公式名称、制限改訂状態）を一括取得するためのエンドポイント。
* **YGOPRODeck 画像サーバー (Image Domain)**: `https://images.ygoprodeck.com/images/cards_small/{card_id}.jpg`
  検索およびデッキに追加された各公式カードの実画像（サムネイル）を取得するためのドメイン。負荷軽減とUXの観点から必ず `cards_small` 階層パスを指定する仕様。

## 3. アプリケーション内ロジック固定値 (app.js)
* `CARD_W_RATIO = 59` : 遊戯王カードの物理的アスペクト比の基準となる横幅比率の基準。
* `CARD_H_RATIO = 86` : 遊戯王カードの物理的アスペクト比の基準となる縦幅比率の基準。（※「単一カードの画像切り抜きモード」を選択した際に、この 59:86 の黄金比率でマウスクロップ枠を強制ロックさせる）
* 切り抜き時の標準保存解像度 (Temp Canvas Constraints):
  - 横: `118 pixel` (59の2倍スケール)
  - 縦: `172 pixel` (86の2倍スケール)
  ※保存領域の軽量化（容量）と、パレットでの表示画質の最適なバランス点として定義。
