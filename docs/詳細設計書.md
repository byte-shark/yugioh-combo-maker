# YGO Combo Maker - 詳細設計書 (Detailed Design)

## 1. アプリケーションステート (State Management)
`app.js` ではアプリケーションの主要な状態が以下のグローバル変数で管理される：
* `cardDatabase` (Array): APIから取得・キャッシュされた全カードオブジェクトのリスト。検索のベースとなる。
* `decks` (Object): 複数デッキを管理するハッシュマップ構造。
  - フォーマット例: `{ "deck_1710000000": { name: "炎王スネークアイ", cards: [CardObject, ImageCardObject, ...] } }`
* `currentDeckId` (String): 現在選択されているアクティブデッキのIDポインタ。
* `routeNodes` (Array): 左側の展開キャンバスに表示されているノード（カードやアクション）の動的配列。都度描画（DOMリフレッシュ）される。

## 2. 主要処理とアルゴリズム
### 2.1 カード検索・表記揺れ吸収モジュール
* `normalizeSearchText(str)`: ユーザーのひらがな入力や英数の全半角文字などを一度カタカナ/半角英数に正規化変換し、APIデータベース内の公式名称と強引にマッチングさせる独自関数。

### 2.2 デッキテキストのパース（自動分離）ロジック
「3x 灰流うらら」等の多種多様なフォーマット・レシピ群から、「カード名」という不純物のない単位だけを抽出する正規表現ロジック。
* Regex: `replace(/^([0-9０-９]+[xXｘＸ枚]?|[xXｘＸ][0-9０-９]+)[\s　]*/, '')` を使用し、文頭の枚数やX表記を完全に削ぎ落としてカード名のSetコレクションを生成。

### 2.3 画像自動切り抜き (Image Crop & Grid Slicer) モジュール
HTML5 Canvas API仕様（`drawImage` と `toDataURL`）を活用した画期的なクライアントサイド高速画像分割・抽出処理。
* **座標スケーリング計算**: CSSによって縮小ブラウザ表示されているCanvasマウスポインタの座標を、`getBoundingClientRect()` からスケール率 (`scaleX`, `scaleY`) を算出してオリジナルの画像ソースのピクセル座標系へ数学的に逆変換（アップスケール）する。
* **グリッド一括カットループ**: ユーザーがGUIから指定した `cols`(列) と `rows`(行) に基づき、囲まれた矩形領域 (`w`, `h`) をforループで等分巡回。`118x172` ピクセルの圧縮用ミニCanvasを経由し、JPEGクオリティ `0.85` の超軽量Base64文字列（1枚あたり約6KB）として抽出し、非同期かつ一括でローカルストレージデッキ内に格納する。

## 3. CSS レイアウト制御 (Scrolling Integrity Limitation)
「Flexboxの一部ブラウザにおける子要素の無限伸長バグ」を完全回避するため、右側のPalette Areaでは `position: absolute` をベースにした絶対座標グリッドで構成。「ヘッダー領域（`top` と `height` 固定）」と「スクロール領域 (`bottom:0`, `overflow-y: auto;`)」 を完全に断絶・分離した構造仕様を適用し、不文の堅牢なスクロール挙動を実現している。
